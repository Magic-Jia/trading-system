## OpenClaw gateway 运维基线更新
- 已确认并修复这台机器上 OpenClaw 双实例反复出现的问题：systemd system service 与 user service 同时 enabled，导致抢占 18789 端口。
- 后续只保留 `openclaw-gateway.service` 这个 systemd system service；user service 必须保持 disabled + inactive。
- 管理 OpenClaw gateway 统一使用 `sudo systemctl` / `sudo journalctl -u openclaw-gateway.service`；不要再把 `openclaw gateway ...` 或 `systemctl --user ...` 当作主入口。
- 当前正确基线：system service = enabled + active；user service = disabled + inactive；`127.0.0.1:18789` 由 system service 持有；`openclaw-gateway` 主进程应只有 1 个。
- 若再次看到两个 OpenClaw 实例，优先检查顺序固定为：system service 状态 → user service 是否被重新启用 → `127.0.0.1:18789` 端口归属；不要再先走 `openclaw gateway ...` 或 `systemctl --user ...` 路径。
