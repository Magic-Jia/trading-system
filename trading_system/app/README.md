# app/ 目录说明

这是自动交易程序的第二阶段骨架目录。

目标：把当前 `trading_system/` 下偏脚本式的能力，逐步迁移为可维护、可测试、可自动执行的模块化程序。

## 模块职责

- `connectors/`：交易所与外部数据接口
- `signals/`：信号生成与策略逻辑
- `risk/`：风险校验、仓位 sizing、熔断限制
- `execution/`：订单生成、执行、重试、防重复
- `portfolio/`：持仓同步、持仓生命周期管理
- `storage/`：运行状态与日志落库
- `reporting/`：日报、统计、异常提醒

## 迁移原则

1. 先保留现有脚本可运行
2. 再把公共逻辑抽到 `app/`
3. 最后让 `main.py` 串起一轮完整自动交易流程

## 当前状态

- 已完成目录骨架
- 下一步优先落地：`risk/` + `execution/` + `storage/`
